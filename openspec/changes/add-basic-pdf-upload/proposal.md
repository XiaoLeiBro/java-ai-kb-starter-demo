## Why

当前 Demo 只支持 Markdown / TXT 上传。真实客户用于试点的制度、客服规则、产品资料更常见的是 PDF 文件。如果演示时要求客户先手动转换为纯文本，会削弱“可交付”的可信度。

本变更只补齐文本型 PDF 的基础解析能力，服务客户演示和体验版交付，不扩展为完整多格式文档中心。

## What Changes

- 后端上传白名单新增 `.pdf`
- 新增文档文本提取组件：
  - `.md` / `.txt` 继续按 UTF-8 文本读取
  - `.pdf` 使用 Apache PDFBox 提取文本
- PDF 解析失败、加密 PDF、空文本 PDF 返回稳定 400 错误
- 前端上传入口仅展示 `.md,.txt,.pdf`
- README、客户介绍和演示脚本同步说明：当前仅支持文本型 PDF，不支持扫描件 OCR

## Non-Goals

- 不支持扫描件 PDF OCR
- 不支持 Word / Excel / PPT
- 不做复杂表格结构还原
- 不做文档异步解析任务中心
- 不做解析进度条、失败重试或批量导入

## Impact

- 新增 Maven 依赖：`org.apache.pdfbox:pdfbox`
- 修改知识库上传索引流程中的文本读取方式
- 修改上传配置默认允许扩展名
- 修改前端文件选择提示
- 增加 PDF 文本提取单元测试
