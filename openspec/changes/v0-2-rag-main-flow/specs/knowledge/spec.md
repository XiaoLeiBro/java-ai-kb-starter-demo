# Knowledge RAG 规格变更

## 新增需求

### 需求：已认证用户应只能管理自己的知识库

系统应当允许已认证用户仅创建和查看自己的知识库。

#### 场景：创建知识库

- **假设** 有一个已认证的用户
- **当** 该用户提交了有效的知识库名称和可选的描述信息
- **那么** 系统返回 HTTP 200 及创建好的知识库
- **并且** 该知识库的所有者为该已认证用户

#### 场景：仅列出当前用户的知识库

- **假设** 有两个已认证用户，各自拥有不同的知识库
- **当** 其中一个用户查看知识库列表
- **那么** 返回结果中仅包含该用户自己的知识库

#### 场景：拒绝未认证用户访问知识库

- **假设** 没有已认证的用户
- **当** 客户端调用知识库相关接口
- **那么** 系统返回 HTTP 401

### 需求：文档上传时应同步索引 Markdown 和 TXT 文件

系统应当接受用户自己知识库中的 Markdown 和 TXT 文件，持久化原始文件、将文本拆分为分块、生成向量嵌入、写入向量记录，并将文档标记为 READY。

#### 场景：上传 Markdown 文档

- **假设** 有一个已认证用户拥有一个知识库
- **当** 该用户上传一个非空的 `.md` 文件
- **那么** 系统将文件存储在 `userId/knowledgeBaseId/documentId/原始文件名` 路径下
- **并且** 存储路径中的 `documentId` 与持久化的文档 ID 一致
- **并且** 系统创建文档分块
- **并且** 系统为这些分块写入向量记录
- **并且** 返回的文档状态为 `READY`

#### 场景：拒绝不支持的文件类型

- **假设** 有一个已认证用户拥有一个知识库
- **当** 该用户上传的文件不是 `.md` 或 `.txt` 格式
- **那么** 系统返回 HTTP 400

#### 场景：上传时隐藏其他用户的知识库

- **假设** 用户 A 拥有一个知识库
- **并且** 用户 B 已认证
- **当** 用户 B 向用户 A 的知识库上传文档
- **那么** 系统返回 HTTP 404

### 需求：向量检索应按知识库隔离

系统应当通过 `knowledgeBaseId` 过滤向量搜索，以确保检索结果不会返回其他知识库的分块。

#### 场景：在两个知识库中搜索相同文本

- **假设** 两个知识库都包含已索引的文档
- **并且** 两者都包含语义上可搜索的文本
- **当** 用户针对其中一个知识库提问
- **那么** 所有返回的引用均属于所请求的知识库
- **并且** 不会返回来自其他知识库的引用

### 需求：聊天应仅基于检索到的引用作答

系统应当验证所有权、对问题生成向量嵌入、从请求的知识库中检索 TopK 条引用，并且仅在存在引用时才调用 LLM。

#### 场景：有引用时进行聊天

- **假设** 有一个已认证用户拥有一个带有已索引分块的知识库
- **当** 该用户提出一个有效的问题
- **那么** 系统从该知识库中检索引用
- **并且** 使用包含引用和问题的提示词调用 LLM
- **并且** 返回 HTTP 200 及 `answer` 和 `references`

#### 场景：无引用时进行聊天

- **假设** 有一个已认证用户拥有一个空知识库，或检索未返回任何分块
- **当** 该用户提出一个有效的问题
- **那么** 系统返回 HTTP 200
- **并且** 答案为 `当前知识库中没有找到相关信息`
- **并且** 引用列表为空
- **并且** 系统不调用 LLM

#### 场景：拒绝无效的聊天请求

- **假设** 有一个已认证用户
- **当** 聊天请求中缺少 `knowledgeBaseId` 或 `question`，或 `topK` 超出范围
- **那么** 系统返回 HTTP 400

#### 场景：聊天时隐藏其他用户的知识库

- **假设** 用户 A 拥有一个知识库
- **并且** 用户 B 已认证
- **当** 用户 B 针对用户 A 的知识库提问
- **那么** 系统返回 HTTP 404
- **并且** 系统不调用 Embedding、向量搜索或 LLM 服务

### 需求：RAG 错误应映射为稳定的 HTTP 响应

系统应当将验证失败、所有权校验失败、LLM、Embedding、向量和文件处理失败映射为稳定的 API 错误，不泄露敏感的底层实现细节。

#### 场景：模型服务商故障

- **假设** 某个请求需要访问 Chat 或 Embedding 服务
- **当** 服务商故障或未配置
- **那么** 系统返回 HTTP 502 及稳定的业务错误码
- **并且** 返回内容不暴露 API 密钥、SQL 或框架堆栈跟踪信息
