# 3 分钟演示脚本

## 演示目标

让潜在客户在 3 分钟内理解：

```text
企业内部资料可以上传到系统，变成一个可提问、可追溯引用、可私有化部署的 AI 知识库助手。
```

演示对象是老板、业务负责人、客服负责人、培训机构负责人或小团队技术负责人，不是专门讲给程序员听。

## 演示前准备

- 启动后端、前端、PostgreSQL、Redis。
- 准备一个演示账号。
- 准备一组样本文档，建议使用 `docs/sample-documents/`。
- 准备 3 个问题：
  - 员工入职需要准备哪些材料？
  - 客户收到商品 8 天后还能退货吗？
  - 产品支持私有化部署吗？
- 不展示真实 API Key、客户资料、数据库密码。

## 0:00～0:20 开场

画面：打开系统首页或知识库列表页。

讲解：

> 这是一个企业知识库 AI 助手 Demo。它可以把企业内部制度、产品资料、客服话术、培训资料上传进来，让员工或客服直接用自然语言提问。系统回答时会给出引用片段，方便追溯答案来自哪份资料。

## 0:20～0:45 登录系统

画面：登录或进入已登录状态。

讲解：

> 系统支持基础账号登录，不同用户只能看到自己的知识库。正式交付时可以根据企业要求接入更完整的权限体系，或者对接企业现有登录系统。

操作：

1. 输入演示账号。
2. 登录进入工作台。
3. 简单展示当前用户信息和健康状态。

## 0:45～1:10 创建知识库

画面：知识库管理页。

讲解：

> 我们可以按业务场景创建不同知识库，比如客服知识库、企业制度知识库、产品资料知识库。这样不同资料不会混在一起，后续也方便做权限和管理。

操作：

1. 创建知识库：`企业制度与客服规则知识库`。
2. 描述：`用于回答员工制度、售后规则、产品部署等问题`。
3. 选择刚创建的知识库。

## 1:10～1:40 上传文档

画面：上传文档区域和文档列表。

讲解：

> 这里上传企业已有资料。Demo 当前支持 Markdown、TXT 和文本型 PDF。扫描件 PDF、Word、Excel、PPT 可以放到商业交付范围，根据客户资料格式继续扩展。上传后系统会自动切分文本、生成向量，并写入向量库。

操作：

1. 上传 `公司年假制度.md`。
2. 上传 `客服售后规则.md`。
3. 上传 `产品介绍.md`。
4. 等待或刷新文档状态为 `READY`。

## 1:40～2:20 AI 问答与引用片段

画面：问答页面。

讲解：

> 现在可以直接用口语化问题查询资料。系统不是凭空回答，而是先从知识库检索相关片段，再让大模型基于这些片段组织答案。

操作和话术：

1. 提问：`客户收到商品 8 天后还能退货吗？`
2. 展示回答。
3. 展示引用片段。

讲解：

> 这里可以看到答案后面有引用来源。业务人员可以检查 AI 是根据哪段资料回答的，避免变成不可追溯的聊天机器人。

继续提问：

1. `员工入职需要准备哪些材料？`
2. `产品支持私有化部署吗？`

讲解：

> 对客服、行政、人事、售前来说，这类重复查询可以节省大量时间，也能统一对外话术。

## 2:20～2:40 对话历史

画面：会话列表或当前会话。

讲解：

> 系统会保留基础对话历史，方便回看之前问过什么、AI 回答了什么。正式交付时可以根据客户要求做更完整的会话管理和审计。

操作：

1. 展示当前会话。
2. 切换或查看历史会话。

## 2:40～2:55 调用记录

画面：调用记录页面。

讲解：

> 每次模型调用都会记录模型、状态、耗时和 Token。这对企业很重要，因为后续可以扩展成本统计、异常排查和审计。

操作：

1. 打开调用记录。
2. 展示最新调用。
3. 强调这不是黑盒聊天，而是可以追踪的企业应用。

## 2:55～3:10 结尾

画面：回到知识库或 README / 演示说明。

讲解：

> 这套 Demo 的核心价值不是做一个通用聊天工具，而是作为 Java 企业 AI 助手的工程模板。它可以私有化部署，也可以基于客户现有 Java 系统二开，比如接入客服系统、文档中心、企业微信、飞书或内部后台。

收尾话术：

> 如果你们有客服话术、制度文档、产品资料或培训资料，可以先拿一小批文档做体验版试点，3～5 天内验证 AI 知识库是否能帮业务减少重复查询和人工答疑。

## 演示注意事项

- 不要承诺当前 Demo 已经支持完整企业权限、多租户、复杂审计。
- 不要把扫描件 PDF OCR、Word、Excel 说成当前免费 Demo 已完整支持。
- 不要展示真实密钥、客户资料、内部合同、财务资料。
- 客户问到大模型是否出网时，要明确：取决于模型配置；完全私有化需要本地模型或企业私有模型服务。
- 客户问价格时，优先推荐体验版，不要一开始推大项目。
