# 免费版 vs 商业版边界

## 核心原则

> **免费版让你相信这个项目是真的；商业版让你能拿它产生结果。**

边界判断：

```text
免费版：能跑通主流程，建立信任
商业版：能二开、能接单、能上线、能交付
企业版：能部署、能定制、能验收、能长期维护
```

免费版不是“残缺版”，而是面向学习和体验的最小闭环。

商业版不是“多几个功能”，而是面向真实项目交付的完整工程能力。

## 商业版本类型

| 版本 | 适合人群 | 内容说明 |
|---|---|---|
| 免费 Demo | 学习、体验、验证方向 | 本仓库提供，当前 v0.3 已跑通用户鉴权、RAG 主流程、对话历史与调用记录 |
| 进阶版 | 想学习完整项目、自己二开 | 完整源码、启动文档、基础二开说明 |
| 专业版 | 接单、公司内部项目、小团队落地 | 多模型适配、Token 统计、调用日志、管理后台、视频讲解 |
| 陪跑版 | 需要部署指导、架构答疑、二开规划 | 专业版内容 + 1v1 架构答疑 + 部署指导 |
| 企业版 | 私有化部署、定制开发、交付验收 | 企业环境部署、功能定制、交付文档、培训支持 |

> 说明：为避免版本差异和服务范围误解，具体费用根据授权范围、服务内容、交付方式和是否需要企业定制单独确认。

## 如何获取商业版本

请通过 [README.md](../README.md) 底部的联系方式联系作者。

联系时建议说明：

1. 预期用途：个人学习 / 接单 / 公司内部项目 / 企业私有化部署
2. 需要的版本类型：进阶版 / 专业版 / 陪跑版 / 企业版
3. 是否需要源码授权、部署指导、二开支持或企业交付
4. 是否需要合同、发票、验收材料
5. 项目周期、团队规模、部署环境

企业客户请补充：

```text
采购流程
合同周期
部署环境
是否内网隔离
是否有等保 / 审计要求
是否需要对接企业微信 / 钉钉 / LDAP / SSO
```

## 授权边界

### 免费 Demo

允许：

```text
学习
研究
本地运行
非商业引用
提交 Issue / 文档类 PR
```

不允许：

```text
直接商用
客户交付
二次销售
去除版权信息
改名后重新发布为同类产品
```

### 商业授权

商业授权会根据版本确认可用范围。

常见授权范围包括：

```text
个人学习
个人二开
接单交付
公司内部项目
企业私有化部署
二次开发与长期维护
```

具体以最终授权说明或合同为准。

## 退款与交付规则

为减少误解，商业版本交付前会确认：

```text
版本内容
授权范围
交付方式
是否包含答疑
是否包含部署
是否包含二开支持
是否包含企业定制
```

一般规则：

- 源码类产品：交付前可协商调整；交付后原则上不支持无理由退款
- 陪跑服务：未开始服务前可协商取消；服务开始后按已发生服务量结算
- 企业版：按双方确认的合同、里程碑和验收规则执行

最终规则以实际沟通确认或合同约定为准。

## 不适合购买商业版的情况

以下情况不建议购买商业版：

```text
只想找一个完全免费的成品系统
不愿意阅读文档
不具备基本 Java / Spring Boot 基础
希望免费版直接用于客户交付
需要的是完整 SaaS，而不是源码和交付模板
对 AI 项目的不确定性完全不能接受
```

商业版更适合：

```text
愿意学习完整工程结构的人
希望二开和接单的人
有公司内部 AI 应用落地需求的人
需要节省调研和踩坑时间的小团队
需要私有化部署和定制交付的企业客户
```
