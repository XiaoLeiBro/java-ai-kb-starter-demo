# 企业知识库 AI 助手介绍

## 一句话介绍

把企业内部文档、制度、产品资料、客服话术和培训资料做成一个可以私有化部署的 AI 问答助手。

员工、客服或管理人员可以直接提问，系统会基于已上传资料回答，并给出引用片段，减少人工查资料和重复沟通成本。

## 适合哪些企业

| 客户类型 | 典型资料 | 可解决的问题 |
|---|---|---|
| 电商团队 | 退换货规则、发货规则、售后话术、产品说明 | 客服重复问题多，新人不熟悉规则 |
| 培训机构 | 课程介绍、报名规则、退款规则、学员 FAQ | 咨询问题重复，课程资料分散 |
| 咨询公司 | 服务方案、交付模板、案例资料、内部 SOP | 顾问查资料慢，知识沉淀难复用 |
| 软件公司 | 产品手册、接口文档、部署说明、客户 FAQ | 售前、售后、实施团队反复查文档 |
| 小微企业 | 规章制度、报销流程、入职指南、合同模板 | 员工找不到制度，行政反复解答 |

当前版本更适合小团队先做试点，不建议一开始用于大型集团、复杂权限组织或强合规生产场景。

## 可以解决哪些问题

- 公司资料散落在文档、群聊、网盘里，新员工不知道从哪里查。
- 客服和销售每天重复回答同类问题。
- 老员工知道规则，但知识没有结构化沉淀。
- 企业想试 AI，但不清楚从哪个业务场景开始。
- 内部资料不希望直接上传到公开 SaaS 平台，希望本地部署或私有化部署。
- 技术团队希望基于 Java 技术栈二开，而不是完全依赖低代码平台。

## 当前系统能做什么

当前公开 Demo 已支持：

- 用户注册、登录和 JWT 鉴权。
- 创建当前用户自己的知识库。
- 上传 Markdown / TXT / 文本型 PDF 文档。
- 本地文件存储。
- 文本切分。
- Embedding 向量化。
- PostgreSQL + pgvector 向量检索。
- 基于知识库内容进行 AI 问答。
- 返回回答和引用片段。
- 创建和查看对话历史。
- 查看基础 LLM 调用记录，包括模型、Token、耗时和状态。
- 使用 Docker Compose 启动后端、前端、PostgreSQL、Redis。

这些能力适合做演示、试点和二开起点。正式商业交付需要根据客户资料格式、部署环境、权限要求和模型供应商继续定制。

## 可定制扩展

根据客户需求，可以在商业交付中扩展：

- 扫描件 PDF OCR / Word / Excel / PPT 文档解析。
- 多用户和基础权限。
- 管理后台。
- Token 用量统计和成本报表。
- 更完整的调用审计。
- DeepSeek、通义千问、硅基流动、Ollama 等模型适配。
- 企业微信、飞书、钉钉入口。
- 与已有 Java 系统、客服系统、文档中心或业务后台集成。
- 完全离线或内网部署方案。

以上能力不是当前免费 Demo 的完整能力，需要按项目范围确认。

## 部署方式

可选部署方式：

| 部署方式 | 适合场景 |
|---|---|
| 本地演示部署 | 快速看效果、试用 Demo |
| 单机 Docker Compose | 小团队试点、内部验证 |
| 客户服务器私有化部署 | 企业资料不出域、内部使用 |
| 定制集成部署 | 接入已有业务系统、统一登录、内部网关 |

第一阶段建议采用单机 Docker Compose 试点，先验证资料问答效果和业务价值，再决定是否做更复杂的企业级部署。

## 数据安全说明

- Demo 默认使用本地文件存储和 PostgreSQL / pgvector 保存业务数据与向量数据。
- 是否调用公网大模型，取决于客户配置的模型供应商。
- 如果需要资料不出内网，需要配置本地模型或企业私有模型服务。
- 不建议在免费 Demo 环境上传客户合同、财务、员工隐私、源代码等敏感资料。
- 商业交付时应明确数据范围、模型供应商、部署网络、账号权限和日志保留策略。

| 版本 | 周期 | 目标 |
|---|---:|---|
| 体验版 | 3～5 天 | 用少量资料部署一套可演示系统 |

## 咨询前需要准备什么

建议提前准备：

- 10～30 份可用于试点的内部文档或客服资料。
- 3～5 个最常见的业务问题。
- 期望部署方式：本地电脑、云服务器、公司内网。
- 是否允许调用公网大模型。
- 是否有现成账号体系、客服系统、文档系统需要对接。
- 期望上线时间和预算范围。

## 常见问题

### 这是通用 SaaS 吗？

不是。当前方向是基于 Java 技术栈的企业 AI 助手工程模板，重点服务私有化部署、二开和项目交付。

### 可以直接用于正式生产吗？

免费 Demo 不建议直接用于正式生产。生产使用需要补齐权限、审计、部署、安全、数据备份、模型成本控制等能力。

### 能不能支持 PDF 和 Word？

当前公开 Demo 支持 Markdown / TXT / 文本型 PDF。扫描件 PDF OCR、Word、Excel、PPT 适合放入商业交付范围，根据客户资料格式定制。

### 数据会不会上传到外部？

业务数据默认存储在部署环境中。AI 回答是否调用外部模型，取决于客户配置。如果要求完全不出内网，需要配置本地模型或企业私有模型服务。

### 和 Dify / FastGPT / MaxKB 有什么不同？

这些平台适合快速搭建 AI 应用。本项目更偏 Java 工程化二开和私有化交付，适合需要接入已有 Java 系统、内部业务规则和企业部署环境的客户。
